Installation without CocoaPods

The Mobile SDK is provided as a dynamic Cocoa Touch Framework. The Mobile SDK depends on the thirdparty frameworks contained in the Thirdparty directory of the SDK package. In the Thirdparty directory you will find three directories:

  • Debug-universal: contains dynamic universal frameworks that can used during development and run on both the simulator and a iOS device and is built with most Debug configurations left to default in Xcode.
  • Release-universial: contains dynamic universal frameworks that can be used during development as well as for Release and run on both the simulator and a iOS device and is built with most Release configurations left to default in Xcode.

When you are developing, drag and drop the following files into your Xcode project:

  • <Mobile Keys SDK package>/Debug/SeosMobileKeysSDK.framework
  • All framework files in <Mobile Keys SDK package>/Thirdparty/Debug-universal

When you are releasing, drag and drop the following files into your Xcode project:

  • <Mobile Keys SDK package>/Release/SeosMobileKeysSDK.framework
  • All framework files in <Mobile Keys SDK package>/Thirdparty/Release-iphoneos

You also have to create a “copy frameworks phase” in Xcode and add the SDK and it’s dependencies there.

If you are not using cocoapods you also need to link against the following iOS libraries

  • Foundation.framework (for basic types)
  • Security.framework (for additional cryptographic functions)
  • CoreMotion.framework (for Twist and Go)
  • CoreLocation.framework (for improved BLE scanning responsiveness)
  • CoreBluetooth.framework (For BLE)
  • CoreTelephony.framework (for reading information from the SIM Card)
  • UIKit.framework
  • SystemConfiguration.framework
  • LocalAuthentication.framework

The thirdparty dependencies used are:

  • JSONModel
  • CocoaLumberjack
  • Mixpanel
  • BerTlv

The Mixpanel dependency also requires you to add the linker flag -licucore

Licensing information about these bundled fameworks is available in the license.plist file that is packaged with the SDK.
