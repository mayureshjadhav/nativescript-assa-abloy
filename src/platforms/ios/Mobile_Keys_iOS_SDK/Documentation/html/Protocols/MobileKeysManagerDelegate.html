  • -mobileKeysShouldInteractWithScannedReader:

    Callback for each advertisement the SDK receives from the readers in the devices vicinity. Return a boolean containing the decision to attempt to open the reader or not. If this callback is implemented, the SDK will call this function on each and every one of the RSSI value measurements, which could impact battery consumption of your application.

    This is a low-level method that should only be implemented for custom opening behaviors, i.e. not one of the types in MobileKeysOpeningType.

    Note

    since version 2.0 @deprecated this delegate callback will be moved to MobileKeysManagerExtendedDelegate
