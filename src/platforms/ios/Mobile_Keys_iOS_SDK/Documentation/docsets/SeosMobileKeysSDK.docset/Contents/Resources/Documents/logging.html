Logging

The Mobile Keys SDK uses a logging framework called CocoaLumberjack. In the Release version of the Mobile Keys SDK this feature is turned off, meaning that no logs are written by the SDK. The Mobile Keys SDK uses different log categories that can be turned on and off individually. To enable a log category, add it as an environment variable in your current scheme in Xcode:

  • Product -> Scheme -> Edit Scheme... -> Select Run in the left menu -> Arguments -> Environment Variables -> (+) -> Name: [One of the categories below] Value: YES

If you want to enable all of the logging, just add the category LogEverything

These are the log categories currently available:

  • LogEverything
  • LogUncategorized
  • LogNotifications
  • LogStatistics
  • LogRSSIScanning
  • LogBeacon
  • LogSeosApdus
  • LogSeosApi
  • LogSeosAccessApi
  • LogApduDesc
  • LogBleCommunication
  • LogBleConnection
  • LogLifecycle
  • LogNetwork
  • LogPersonalization
  • LogAPI
  • LogSettings
  • LogOpening
  • LogFileSystem

You also need to enable one of the CocoaLumberjack loggers:

Add the following import statements to your AppDelegate class

#import <CocoaLumberjack/DDLog.h>
#import <CocoaLumberjack/DDTTYLogger.h>
#import <CocoaLumberjack/DDASLLogger.h>

Add the following method calls to your application:didFinishLaunchingWithOptions: method in your AppDelegate

[DDLog addLogger:[DDTTYLogger sharedInstance]];
[DDLog addLogger:[DDASLLogger sharedInstance]];
